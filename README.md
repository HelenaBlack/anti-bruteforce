# Сервис Анти-Брутфорс

Микросервис, предназначенный для предотвращения атак методом перебора (brute-force) при авторизации. Он ограничивает частоту попыток на основе логина, пароля и IP-адреса.

## Возможности
- Ограничение частоты запросов с использованием Sliding Window (Redis)
- Черные и белые списки IP-адресов (PostgreSQL CIDR)
- CLI для администрирования
- gRPC API

## Требования
- Go 1.22+
- Docker и Docker Compose
- Protoc (для генерации API)

## Быстрый запуск

### 1. Запуск сервиса
```bash
make run
```
Эта команда запустит PostgreSQL, Redis и сам сервис Анти-брутфорс.

### 2. Конфигурация
Конфигурация управляется через переменные окружения (см. `docker-compose.yaml` или `.env`):
- `APP_LIMIT_N`: Макс. количество попыток для логина в минуту (по умолчанию 10)
- `APP_LIMIT_M`: Макс. количество попыток для пароля в минуту (по умолчанию 100)
- `APP_LIMIT_K`: Макс. количество попыток для IP в минуту (по умолчанию 1000)

### 3. Использование CLI
Соберите CLI:
```bash
go build -o admin ./cmd/admin
```

Добавьте IP в черный список:
```bash
./admin blacklist add 192.168.1.0/24
```

Сбросьте бакеты для пользователя:
```bash
./admin reset --login user1 --ip 1.1.1.1
```

## Методы API

### `Check(login, password, ip)`
Проверяет, разрешена ли попытка авторизации.
1. Если IP в белом списке -> OK
2. Если IP в черном списке -> Отказано
3. Если превышены лимиты (N, M, K) -> Отказано
4. Иначе -> OK

### `Reset(login, ip)`
Очищает бакеты для данного логина и IP.
